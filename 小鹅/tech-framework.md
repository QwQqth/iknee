## 二、核心功能 1：AI 语音交互

### 功能定位
通过语音 + 文字双通道，辅助老年用户使用小程序，降低操作门槛。

### 技术选型

```
用户按住说话 ──▶ wx.getRecorderManager() 录音(mp3/16kHz)
       │
       ▼
上传录音文件 ──▶ 后端 /asr 接口 ──▶ 腾讯云语音识别 ASR
       │                                    │
       │                                    ▼
       │                              返回识别文字
       │                                    │
       ▼                                    ▼
   识别文字 ──▶ NLU 意图理解 ──▶ 生成回复文字
                                     │
                                     ▼
                              后端 /tts 接口 ──▶ 腾讯云语音合成 TTS
                                     │
                                     ▼
                              返回音频 URL ──▶ InnerAudioContext 播放
```

| 模块 | 方案 | 说明 |
|------|------|------|
| **录音** | `wx.getRecorderManager()` | 微信原生，mp3 格式 16kHz 采样率，**已实现** |
| **语音识别 ASR** | 腾讯云语音识别（一句话识别/实时流式） | 后端调 API，小程序上传录音文件；也可考虑微信同声传译插件（但类目限制） |
| **语音合成 TTS** | 腾讯云语音合成 | 后端调 API 返回音频 URL；选"成年女性"音色，温暖亲切 |
| **意图理解 NLU** | 方案 A：规则引擎（关键词匹配）<br>方案 B：腾讯云 NLP / 大模型 | 早期用规则引擎（成本低、可控），后期可升级大模型 |
| **前端工具层** | `utils/voice.js`（**已实现**） | 封装录音/识别/合成/播放全流程，模拟模式可独立开发 |

### NLU 意图映射（规则引擎阶段）

```
用户语音 → 识别文字 → 关键词匹配 → 意图 → 执行动作

示例：
"我想做测评"     → 测评意图 → 跳转测评页
"今天练什么"     → 训练意图 → 推荐每日训练
"我的膝盖怎么样" → 档案意图 → 展示健康数据
"我不会用"       → 帮助意图 → 语音引导操作步骤
"播放视频"       → 视频意图 → 跳转科普视频
```

### 关键约束
- **隐私合规**：录音前必须获取用户授权（`wx.authorize({ scope: 'scope.record' })`）
- **模拟模式**：当前 `voice.js` 已实现模拟模式，后端就绪后只需填入 `API_BASE`

## 三、核心功能 2：运动动作识别（最核心）

### 功能定位
拍摄老人运动过程，实时检测动作是否标准，同步语音提示与鼓励，记录训练数据。

### 技术选型对比

| 方案 | 优点 | 缺点 | 适合度 |
|------|------|------|--------|
| **A. 微信 VisionKit Body 检测** | 原生能力、零成本、实时帧级检测、2D/3D 关键点 | 只有 23 个关键点，无内置运动判定逻辑，需自研算法 | ⭐⭐⭐⭐ |
| **B. AI运动识别小程序插件**（云智等） | 内置 10+ 运动识别、集成快、有姿态判定 | 第三方依赖、需付费、膝关节专项动作可能不在列表中 | ⭐⭐⭐ |
| **C. 后端视频分析** | 可用高精度模型、不受小程序算力限制 | 非实时、延迟高、带宽成本高、隐私风险 | ⭐⭐ |
| **D. TensorFlow.js + MoveNet** | 模型开源、可自定义 | 小程序 tfjs 插件兼容性差、性能不稳 | ⭐⭐ |

### 推荐方案：A + 自研运动判定（VisionKit + 规则引擎）

**理由**：
1. 微信原生 VisionKit 提供实时 23 点人体关键点，覆盖膝关节所需的所有关键关节
2. 膝关节运动是**限定动作集**（如直腿抬高、坐姿屈伸、靠墙半蹲等），不需要通用动作识别，用规则引擎判定比通用模型更精准
3. 零第三方依赖，长期可控
4. 如果后续发现关键点不够用，可开启 3D 模式（SMPL-24 点）

### VisionKit 关键点 → 膝关节动作映射

```
VisionKit 23 点中，膝关节运动核心关键点：

编号  名称          用途
──────────────────────────────────
 11   左髋 (L_Hip)    ┐
 12   左膝 (L_Knee)   ├─ 计算左侧膝关节角度
 13   左踝 (L_Ankle)  ┘
 14   右髋 (R_Hip)    ┐
 15   右膝 (R_Knee)   ├─ 计算右侧膝关节角度
 16   右踝 (R_Ankle)  ┘
  5   左肩 (L_Shoulder) ┐
  7   左髋 (L_Hip)      ├─ 辅助判断躯干姿态
 11   左髋 (L_Hip)      ┘

膝关节角度计算公式：
angle = arccos(
  (vec(knee→hip) · vec(knee→ankle)) / 
  (|vec(knee→hip)| × |vec(knee→ankle)|)
)
```

### 膝关节专项动作模板（示例）

```
动作：直腿抬高 (Straight Leg Raise)
┌─────────────────────────────────────────┐
│ 标准姿势：仰卧，一条腿伸直抬起 30-45°    │
│                                         │
│ 检测逻辑：                               │
│ 1. 起始位：膝关节角 ≈ 180°（腿伸直）     │
│ 2. 抬腿中：髋-膝角度从 90° → 45°        │
│ 3. 膝关节角全程 ≥ 160°（腿不弯曲）       │
│                                         │
│ 评分规则：                               │
│ - 膝关节角 < 150° → "腿伸直一些"         │
│ - 抬腿角度 < 30° → "再抬高一点"          │
│ - 全程标准 → "非常好！保持住"             │
└─────────────────────────────────────────┘

动作：靠墙半蹲 (Wall Sit)
┌─────────────────────────────────────────┐
│ 标准姿势：背靠墙，大腿与地面平行          │
│                                         │
│ 检测逻辑：                               │
│ 1. 膝关节角 ≈ 90° ± 15°                 │
│ 2. 大腿与小腿夹角 90° 左右               │
│ 3. 躯干竖直（肩-髋连线与地面垂直）        │
│                                         │
│ 评分规则：                               │
│ - 膝关节角 > 120° → "蹲低一些"           │
│ - 膝关节角 < 70° → "不要蹲太深，伤膝盖"  │
│ - 膝盖内扣（两膝间距 < 踝间距）→ 提醒    │
└─────────────────────────────────────────┘
```

### 实时语音提示策略

```
触发条件                    → 语音提示内容
──────────────────────────────────────────
动作开始（检测到起始位）    → "好的，开始！跟着我一起做"
动作标准                   → "做得很好！""很棒，继续保持"
动作偏差（轻微）           → "腿再伸直一点""膝盖不要内扣"
动作偏差（严重）           → "注意姿势，这样容易受伤，我来教您"
动作完成（计数 +1）        → "完成了！非常棒！已经做了 X 次"
训练结束                   → "今天的训练完成啦！您表现得很棒！"
```

### 关键约束
- **类目要求**：VisionKit Body 检测需要小程序类目支持（AR 类或工具类），公益类可能需单独申请
- **基础库版本**：需 ≥ 2.28.0，3D 关键点需微信 ≥ 8.1.0
- **性能**：实时检测对中低端机型有压力，需降帧策略（如 10fps 检测）
- **隐私**：摄像头使用需用户授权 + 明确告知用途

### 存储方案

| 数据类型 | 存储方式 | 说明 |
|---------|---------|------|
| **用户信息** | 云数据库 `users` | openid 为主键，关联健康档案 |
| **健康档案** | 云数据库 `health_files` | 膝关节评级（如：健康/轻度风险/中度/需就医）、测评历史、风险标签 |
| **训练记录** | 云数据库 `training_records` | 每次训练的动作、次数、评分、角度偏差等详细数据 |
| **测评记录** | 云数据库 `assessments` | 问答式测评的答案和评级结果 |
| **科普视频** | 云存储 (COS) + 云数据库 `videos` | 视频文件存 COS，元数据存 CloudDB |
| **训练快照** | 云存储 (COS) | 用户训练时的关键帧截图（可选，用于回顾） |

### 后端方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **A. 微信云开发** | 零运维、天然鉴权、CloudDB + 云函数 + 云存储一体化 | 冷启动延迟、复杂查询受限 | ⭐⭐⭐⭐⭐ |
| **B. 自建服务器（腾讯云 CVM/轻量）** | 完全控制、性能稳定 | 运维成本高、需自建鉴权 | ⭐⭐⭐ |
| **C. Serverless（腾讯云 SCF + TDSQL）** | 弹性伸缩、按量付费 | 调试复杂、冷启动 | ⭐⭐⭐ |

### 推荐：微信云开发

**理由**：
1. 小程序天然鉴权（openid 自动获取，无需自建登录）
2. CloudDB 前端可直接调用，减少中间层
3. 云存储处理视频文件，自带 CDN
4. 公益项目用量不大，免费额度足够初期使用
5. 语音 ASR/TTS 可通过云函数调腾讯云 API

## 七、风险与待确认项

| 风险项 | 影响 | 应对 |
